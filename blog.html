*Discuss the scope of var, let, and const

overview of the scope of var, let, and const in JavaScript:

  • 1. var
    • i.Function-scoped: Variables are accessible within the function they are declared in.
    • ii.Hoisting: Variables are hoisted to the top of their containing function.
    • iii.No block scope: Variables are not limited to the block they are declared in.
  • 2. let
    • i.Block-scoped: Variables are accessible only within the block they are declared in.
    • ii.Hoisting: Variables are hoisted within the block but not initialized until their declaration.
    • iii.Reassignment: Variables can be reassigned after their initial declaration.
  • 3. Const
    • i.Block-scoped: Variables are accessible only within the block they are declared in.
    • ii.Hoisting: Variables are hoisted within the block but not initialized until their declaration.
    • iii.Immutable values: Variables cannot be reassigned once they are initialized. However, for objects and arrays, their properties or elements can be modified.

*Tell us the use cases of null and undefined

Here are the common use cases of null and undefined in JavaScript:

  • 1. undefined
    • i.Default values for uninitialized variables.
    • ii.Return values from functions that don't explicitly return anything.
    • iii.Indicating the absence of a property in an object.
  • 2. Null
    • i.Explicitly indicating that a variable or object property has no value.
    • ii.Resetting a variable or property to an empty state.
    • iii.Checking if a variable has been intentionally set to no value in conditional statements.

*What do you mean by REST API?

In short, a REST API (Representational State Transfer Application Programming Interface) is a set of rules and conventions for building and interacting with web services using standard HTTP methods (GET, POST, PUT, DELETE). It allows clients (e.g., web browsers or mobile apps) to request and manipulate data from a server in a standardized way, typically using JSON or XML for data representation. REST APIs are stateless, meaning each request from a client to a server contains all the information needed, and they are widely used for web and mobile application development.
